I'm happy to welcome you to the new OpenScan blog!

We moved the OpenScan blog to a new home: [blog.openscan.eu](https://blog.openscan.eu/) and changed it from a Shopify blog to a Jekyll-based static site.


## Why we moved

Though Shopify is a great platform for it is not ideal for our needs. We needed a simple, yet powerful, blog system which handles technical content well. 

With the new setup we can:

- edit posts as plain Markdown
- keep changes in Git
- review migration changes more easily
- preserve older articles without mixing them into shop infrastructure
- adjust the layout and design without touching product pages

We [promised](https://blog.openscan.eu/posts/new-years-recap-and-outlook/) in the beginning of this year, that we wanted to be more communcative about OpenScan and its future. The new blog is a step in that direction.

## What's new?

We hope that you like the new layout and design and find it easier to read too! There are some quality of life improvements like a dark mode, proper tagging and categorization and a respnsive layout for mobile devices.

Some older posts are marked with an archive note. Those articles are still useful as project history, but they may describe older hardware, older software, or workflows that have changed since publication.

If you notice a broken link, missing image, or odd formatting issue, please let us know.

## Thanks to Jekyll and Chirpy

This new version is built with [Jekyll](https://jekyllrb.com/) and the [Chirpy theme](https://github.com/cotes2020/jekyll-theme-chirpy). 

Jekyll gives us a simple static-site workflow that fits the blog well: posts are files, builds are reproducible, and the generated site is easy to deploy. Chirpy provides a solid base theme with the navigation, archives, tags, search, responsive layout, and dark mode already handled in a clean way.

We adjusted the theme to better match OpenScan, but the foundation is still Chirpy. Thanks to everyone maintaining these open-source tools.
